목록으로

  • XML(Extensible Markup Language)

    W3C에서 개발된 다목적 마크업 언어(태그 등을 이용하여 문서나 데이터의 구조를 명기하는 언어의 한 가지, 일반적으로 데이터를 기술하는 정도로만 사용된다)이다.
    XML은 주로 다른 종류의 시스템, 특히 인터넷에 연결된 시스템끼리 데이터를 쉽게 주고 받을 수 있게 하여 HTML의 한계를 극복할 목적으로 만들어졌다.
    이런식으로 HTML과 굉장히 비슷한 형태를 띄고 있다. (사실 위의 그림이 제대로 된 XML인지 모르겠다. XML과 HTML의 차이에 대해서도 추후에 알아볼 예정이다.)

  • JSON(JavaScript Object Notation)

    속성-값 쌍으로 이루어진 데이터 오브젝트를 전달하기 위해 텍스트를 사용하는 개방형 표준 포맷이다.
    비동기 비라우저/서버 통신을 위해 XML을 대체하는 주요 데이터 포멧이다.
    자바스크립트 언어로부터 파생되어 자바스크립트의 구문 형식을 따르지만 언어 독립형인 데이터 포맷이다.
    그렇기 때문에 프로그래밍 언어나 플랫폼에 독립적이므로 수 많은 프로그래밍 언어에서 쉽게 이용할 수 있는 언어이다.
    위의 XML의 내용을 JSON으로 옮기면 이렇게 된다.

  • CSON(CoffeeScript Object Notation)

    CSON또한 데이터 포멧의 한 가지로써 JSON을 사용함에 있어서 불편한 점 몇가지를 보완한 데이터 포멧이라고 할 수 있다.
    CSON이 JSON과 다른점은 다음과 같다.

    1. CSON에서는 주석을 입력할 수 있다.
    2. CSON에서 배열을 입력할 때 콤마(,)로 구분하지 않는다.
    3. CSON은 개행을 구분하기 위해서는 \n을 입력하지 않아도 된다. 입력시 구분 해놓은 대로 그대로 들어간다.
      JSON에서는 엔터를 아무리 눌러도 개행을 해주기 위해서는 \n을 입력하여 집어넣어야한다.
    4. 객체를 표현할 때 {}를 입력하지 않아도 된다.

    CSON의 정확한 표기법을 몰라 GitHub의 CSON에서 사진을 가져왔다.
