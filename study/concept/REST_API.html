  • REST API

    REST API에 대한 설명은 첨부한 링크중에 첫 번째 링크로 대체하도록 하겠다. 필자는 첫 번째 링크를 보고 REST API의 대부분을 이해했다.

    하지만 최근 쿠키와 세션을 공부하고 나서 REST API의 내용을 읽고 나니, REST API의 특징중 Stateless 쪽에서 생기는 의문점이 있어서 그 부분을 집고 넘어가겠다.
    stateless 라면서 세션을 사용하지 않는다고 한다. 그렇다면 클라이언트에 대한 처리는 어떻게 하는 것일까? 권한 확인이라던지 말이다. 첨부한 마지막 링크에서는 인증(Authenication)과 인가(Authentication)을 나누어서 설명한다. 하지만 예시의경우 다른 포털들의 API를 사용할 때를 예시를 들고 있기 때문에 내가 궁금한 '나의 서비스를 REST API의 틀에 맞추어서 만들면 인증과 인가는 어떻게 확인해야하는가?'는 해결되지 않은듯 하다. 추후에 알아보아야할듯 하다.
