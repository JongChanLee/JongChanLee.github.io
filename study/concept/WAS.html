목록으로

  • 웹 서버

    웹 서버는 http요청에 대한 응답으로 정적인 웹페이지를 반환해주는 서버를 말한다. 그렇다면 정적인 웹페이지라는 것은 뭘까? 정적인 웹페이지는 사용자의 입력 등으로 인해 html 문서의 내용이 바뀌지 않는 웹페이지를 말한다. 지금 나의 이 페이지를 보고 있다면 이 웹페이지는 정적인 웹페이지이다. 댓글을 쓸 수 있는 공간이 없을 뿐더러 내가 저장해놓은 문서 그대로를 보여주기만 하기 때문이다. 정적인 웹페이지는 그런 것을 가리킨다.

    웹서버는 이렇게 사용자의 입력 또는 다른 요인들에 의해 실시간으로 바뀌는 웹페이지를 생성하는 것이 아닌, 있는 그대로의 문서만을 보내주는 서버를 말한다.

    대표적인 예로 '아파치 서버'가 있다.

  • 웹 어플리케이션 서버

    Web Application Server(WAS라고 한다.)는 웹 서버와 반대로 동적인 웹페이지를 반환해주는 서버를 말한다. 동적인 웹페이지는 정적인 웹페이지와 반대로 실시간으로 사용자의 입력 등의 요인에 따라 변하는 웹페이지를 말한다.

    네이버 검색창에 검색한 키워드에 따라 검색 결과가 바뀌는 것이 동적인 웹페이지이다.

    WAS 안에는 웹 서버와 함께 동적인 데이터의 처리를 해주는 웹 컨테이너가 들어간다.(Web Container) 위 그림에서 웹 어플리케이션 서버 안에 있는 웹 서버가 클라이언트의 요청을 받으면 동적인 데이터의 처리에 대한 처리를 웹 컨테이너에게 요청한다. 웹 컨테이너는 동적인 데이터를 처리한 후 그 결과를 웹 서버에게 돌려준다. 웹서버는 이를가지고 최종 html파일을 만들어서 클라이언트에서 응답해준다.

  • 아파치와 톰캣

    • 아파치(Apach) 서버
      • 아파치 서버는 아파치 재단에서 후원하고 있는 오픈 소스 프로젝트 커뮤니티에서 만든 http 웹 서버를 지칭하는 말이다.
      대부분의 운영체제에서 실행될 수 있으며, 가장 널리 쓰인다.
    • 톰캣(Tomcat)
      • 톰캣은 마찬가지로 아파치 재단의 프로젝트 중 하나이며, 웹 어플리케이션 서버로서 자바 서블릿을 실행시키고 JSP 코드가 포함되어 있는 웹페이지를 만들어준다. 즉 톰캣은 웹 어플리케이션 서버이며 java로 되어있는 서버만 돌릴 수 있다.

  • 서버 정리?

    웹 서비스를 만든다고해서 아파치 라던지 다른 여러 웹서버 또는 웹 어플리케이션 서버를 사용할 필요가 없다. 개발하려는 서버측의 언어와 프레임워크에 따라서 서버에 대한 처리를 알아서 해주는 것도 있습니다.
    예를 들어서 node.js 를 사용하여 웹 서비스를 만들거나 Spring 프레임워크를 사용해서 만들면 톰캣과 같은 웹 어플리케이션 서버가 따로 필요가 없다. 프레임워크에서 서버에 대한 처리를 알아서 해주기 때문이다.(http 요청에 대한 처리)
